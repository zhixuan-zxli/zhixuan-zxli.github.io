vLLM frontend to backend: AsyncLLM and Engine
2026-07-14 00:00 label vLLM  

分解 vLLM 前端 (AsyncLLM) 和 后端 (EngineCore) 架构。 综合自 博客 Anatomy of vLLM（Aleksa Gordic，2025-09-05）、微信公众号文章 透过 vLLM 深入分析一次推理的完整旅程，以及 vLLM 源码分析（基于 releases/v0.14.0 分支）。

1. HTTP 入口：FastAPI → OpenAIServing → AsyncLLM

1.1 FastAPI Route (vllm/entrypoints/openai/api_server.py:478)

@router.post("/v1/chat/completions", dependencies=[Depends(validate_json_request)])

@load_aware_call
async def create_chat_completion(request: ChatCompletionRequest, raw_request: Request):

1.2 OpenAIServingChat (vllm/entrypoints/openai/serving_chat.py:224)

预处理分为两个阶段：chat template 渲染 + tokenize（_preprocess_chat）和构造引擎请求（_process_inputs）。

async def create_chat_completion(self, request, raw_request=None):
    # 1) 模型检查 + tokenizer 获取
    tokenizer = await self.engine_client.get_tokenizer()
    # 2a) 预处理阶段一：渲染 chat template + 分词
    conversation, engine_prompts = await self._preprocess_chat(
        request, tokenizer, request.messages, ...)
    # 2b) 预处理阶段二：封装为 EngineCoreRequest + 送入引擎
    for i, engine_prompt in enumerate(engine_prompts):
        engine_request, tokenization_kwargs = await self._process_inputs(
            request_id, engine_prompt, sampling_params, ...)
        generator = self.engine_client.generate(
            engine_request, sampling_params, request_id, ...) # type: AsyncGenerator[RequestOutput, None]
    # 3) 包装为 OpenAI 格式的 Response / SSE Stream
    return await self.chat_completion_full_generator(request, result_generator, ...)

  • _process_inputs() — TokensPrompt → EngineCoreRequest (serving_engine.py:1293)

2. AsyncLLM：前端进程的指挥中心 (vllm/v1/engine/async_llm.py)

2.1 构造函数

class AsyncLLM(EngineClient):
    def __init__(self, vllm_config, executor_class, log_stats, ...):
        # 1) Tokenizer
        self.tokenizer = cached_tokenizer_from_config(model_config)

        # 2) InputProcessor：原始输入 → EngineCoreRequest
        self.input_processor = InputProcessor(vllm_config, tokenizer)

        # 3) OutputProcessor：EngineCoreOutputs → RequestOutput
        self.output_processor = OutputProcessor(tokenizer, log_stats, ...)

        # 4) EngineCoreClient：跨进程通信桥梁（InprocClient / AsyncMPClient / DPLBAsyncMPClient）
        self.engine_core = EngineCoreClient.make_async_mp_client(vllm_config, executor_class, ...)

        # 5) 启动 output_handler 后台 asyncio task
        self._run_output_handler()

2.2 generate()：核心生成流程

async def generate(self, prompt, sampling_params, request_id, ...):
    # 1) 创建 RequestOutputCollector（per-request async queue）
    q = await self.add_request(request_id, prompt, sampling_params, ...)

    # 2) 循环从 queue 中拉取 RequestOutput，yield 给上层
    finished = False
    while not finished:
        out = q.get_nowait() or await q.get()
        finished = out.finished
        yield out  # RequestOutput

2.3 add_request()：请求入队

async def add_request(self, request_id: str, prompt: EngineCoreRequest, ...):
    #
    request = self.input_processor.process_inputs(request_id, prompt, ...)
    self._run_output_handler()
    # 注册到 OutputProcessor（前端进程）
    self.output_processor.add_request(request, prompt, parent_req, index, queue)
    # 发送到 EngineCore（后端进程，ZMQ）
    await self.engine_core.add_request_async(request)
    #
    return queue # type: RequestOutputCollector

请求被同时注册到两个位置：OutputProcessor（前端，管输出）和 EngineCoreClient（后端，管调度执行）。

2.4 output_handler：后台输出泵

async def output_handler():
    while True:
        # 1) 从 EngineCore（ZMQ output socket）拉取 EngineCoreOutputs
        outputs = await engine_core.get_output_async()

        # 2) 分块处理（避免长时间阻塞 event loop）
        for start in range(0, num_outputs, chunk_size):
            outputs_slice = engine_core_outputs[start:end]
            processed = output_processor.process_outputs(outputs_slice, ...)
            # RequestOutputs 被推送到对应 request 的 queue

            # 3) 终止需要 abort 的请求（stop string 命中）
            if processed.reqs_to_abort:
                await engine_core.abort_requests_async(processed.reqs_to_abort)

3. EngineCoreClient：跨进程通信层 (vllm/v1/engine/core_client.py)

3.1 类继承体系

EngineCoreClient (ABC)
├── InprocClient          # 同进程模式（V0 兼容 / 调试用）
│     └── 直接持有 EngineCore 实例，无 ZMQ
├── MPClient (ABC)        # 多进程基类（ZMQ 通信）
│     ├── SyncMPClient    # 同步模式（LLM.generate 离线推理）
│     └── AsyncMPClient   # 异步模式（AsyncLLM 在线服务）
│           ├── DPAsyncMPClient    # 数据并行 + 外部负载均衡
│           └── DPLBAsyncMPClient  # 数据并行 + 内部负载均衡

3.2 MPClient：多进程通信基类

class MPClient(EngineCoreClient):
    def __init__(self, asyncio_mode, vllm_config, executor_class, log_stats, ...):
        # 序列化
        self.encoder = MsgpackEncoder()
        self.decoder = MsgpackDecoder(EngineCoreOutputs)

        # 启动 EngineCore 进程（通过 launch_core_engines）
        engine_manager, coordinator, addresses = launch_core_engines(...)

        # 创建 ZMQ sockets
        self.input_socket = make_zmq_socket(ctx, input_address, zmq.ROUTER, bind=True)
        self.output_socket = make_zmq_socket(ctx, output_address, zmq.PULL)

3.3 AsyncMPClient：生产环境的异步客户端

class AsyncMPClient(MPClient):
    def __init__(self, ...):
        self.outputs_queue = asyncio.Queue[EngineCoreOutputs | Exception]()
        self._ensure_output_queue_task()  # 后台 asyncio task 从 output_socket 读取

    async def get_output_async(self) -> EngineCoreOutputs:
        """被 AsyncLLM.output_handler 调用"""
        return await self.outputs_queue.get()

    async def add_request_async(self, request: EngineCoreRequest) -> None:
        """发送 ADD 请求到 EngineCore"""
        await self._send_input(EngineCoreRequestType.ADD, request)

3.5 DPCoordinator (vllm/v1/engine/coordinator.py)

DPCoordinator（独立进程，DP > 1 时启动）
├── 收集各 DP engine rank 的 stats（waiting/running counts）
├── 广播 stats 给所有 frontend process（用于 LB 决策）
├── 追踪 "request wave" 和 running state
└── 收到 FIRST_REQ 消息时广播 START_DP_WAVE 唤醒 engines

4. EngineCore & EngineCoreProc：后端进程的引擎核心 (vllm/v1/engine/core.py)

类层次

EngineCore                      # 引擎核心逻辑（调度 + 执行 + 后处理）
└── EngineCoreProc              # ZMQ 封装 → 作为独立进程运行
     └── DPEngineCoreProc       # DP并行 / MoE

EngineCoreProc：ZMQ 封装的进程模式

class EngineCoreProc(EngineCore):
    """ZMQ-wrapper for running EngineCore in background process."""

    def __init__(self, vllm_config, handshake_address, executor_class, ...):
        # 三队列架构
        self.input_queue = queue.Queue()    # 接收来自 input thread 的请求
        self.output_queue = queue.Queue()   # 输出到 output thread

        # 启动 handshake（ZMQ DEALER）
        with self._perform_handshakes(...) as addresses:
            # 初始化父类 EngineCore
            super().__init__(vllm_config, executor_class, ...)

            # input thread：ZMQ input socket → input_queue
            input_thread = threading.Thread(target=self.process_input_sockets, ...)

            # output thread：output_queue → ZMQ output socket
            self.output_thread = threading.Thread(target=self.process_output_sockets, ...)
            
    def run_engine_core(*args, dp_rank: int = 0, local_dp_rank: int = 0, **kwargs):

        engine_core: EngineCoreProc | None = None

        if data_parallel and vllm_config.model_config.is_moe:
            # Set data parallel rank for this engine process.
            parallel_config.data_parallel_rank = dp_rank
            engine_core = DPEngineCoreProc(*args, **kwargs)
        else:
            engine_core = EngineCoreProc(*args, engine_index=dp_rank, **kwargs)

        engine_core.run_busy_loop()

    def run_busy_loop(self):
        while True:
            # 1) 从 input_queue 拉取请求，加入 scheduler
            self._process_input_queue()
            # 2) 执行一步引擎循环
            self._process_engine_step()

5. Executor → Worker ：模型执行层 (vllm/v1/executor/)

5.1 Executor 抽象

class Executor(ABC):
    """An executor is responsible for executing the model on one or more devices."""

    @staticmethod
    def get_class(vllm_config) -> type["Executor"]:
        if backend == "mp":
            return MultiprocExecutor  # 多 GPU（TP/PP），本地 multiprocessing
        elif backend == "ray":
            return RayDistributedExecutor  # 分布式，Ray 编排

    @abstractmethod
    def execute_model(self, scheduler_output, non_block=False):
        """执行一次模型前向 + 采样"""
    @abstractmethod
    def collective_rpc(self, method, timeout, args, kwargs):
        """在所有 worker 上执行 RPC 调用"""

Executor 对比：

Executor Workers 通信方式 适用场景
UniProcExecutor 1 个 Worker（同进程） 直接调用 单 GPU 调试 / 小模型
MultiprocExecutor N 个 Worker（独立进程） rpc_broadcast_mq（共享内存队列） 单机多 GPU（TP/PP）
RayDistributedExecutor N 个 Ray Actor Ray 多机分布式

5.2 MultiprocExecutor：多 GPU 编排

class MultiprocExecutor(Executor):
    def __init__(self, vllm_config):
        # 为每个 rank 创建 WorkerProc 子进程
        for rank in range(world_size):
            WorkerProc.make_worker_process(rank, ...)

    def execute_model(self, scheduler_output, non_block=False):
        return self.collective_rpc(
            "execute_model", 
            args=(scheduler_output),
            ...
        )
